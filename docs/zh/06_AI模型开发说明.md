# MYTutor - AI 模型开发说明

> 最后更新：2026-06-13

## AI 集成概览

MYTutor 通过**火山方舟（Volcano Ark）**——字节跳动的 AI 平台——集成智能辅导功能。

---

## 1. AI 对话辅导（火山方舟）

### 模型信息
- **服务商**：火山方舟（字节跳动）
- **模型**：DeepSeek / 豆包系列
- **集成点**：`com.yy.my_tutor.ark.service.ArkAIService`

### 功能
- 上下文感知的数学辅导对话
- 多轮对话，支持历史记录
- 图片理解（对话中上传图片）
- 消息加密存储

### 架构

```
用户消息 → ArkClientController → ArkAIService → 火山方舟 API
                ↑                       ↓
          ChatMessageMapper      AI 响应（流式）
                ↑                       ↓
          MySQL（加密存储）       用户收到响应
```

### 关键实现细节
- **上下文管理**：加载历史消息作为上下文，保持对话连续性
- **流式响应**：响应逐 Token 流式传输，实时显示
- **加密存储**：对话消息加密后存储到数据库
- **文件上传**：图片上传并作为对话上下文处理
- **大文件支持**：流式上传，支持 500MB 文件（无 OOM）

### API 端点
| 端点 | 方法 | 描述 |
|------|------|------|
| `/ark/chat` | POST | 发送消息给 AI 辅导老师 |
| `/ark/upload` | POST | 上传文件/图片供 AI 分析 |

## 2. AI 智能出题

### 模型信息
- **服务商**：火山方舟
- **集成点**：`com.yy.my_tutor.test.service.impl.AITestGenerateServiceImpl`

### 功能
- 基于知识点生成练习题
- 自适应难度调整
- 多种题型（选择题、填空题、解答题）
- 双语内容（英语/法语）

### 出题流程

```
1. 学生选择知识点
2. 系统查询知识点详情 + 掌握程度
3. AI 根据以下条件生成题目：
   - 知识点内容
   - 学生当前掌握水平
   - 目标难度分布
4. 题目存入题库
5. 组卷：按难度均衡选题
```

### 题型
| 类型 | 编码 | 描述 |
|------|------|------|
| 选择题 | 1 | 4个选项，单选正确答案 |
| 填空题 | 2 | 需要简短作答 |
| 解答题 | 3 | 需要分步解答 |

### 难度等级
| 等级 | 编码 | 描述 |
|------|------|------|
| 简单 | 1 | 基础概念应用 |
| 中等 | 2 | 多步骤问题解决 |
| 困难 | 3 | 复杂推理与综合运用 |

### 自适应出题
- **集成点**：`com.yy.my_tutor.test.service.impl.AdaptiveTestServiceImpl`
- 根据学生历史表现调整难度
- 利用 `knowledge_mastery` 表确定合适难度
- 按难度等级均衡分配题目

## 3. AI 课程生成

### 模型信息
- **服务商**：火山方舟
- **集成点**：`com.yy.my_tutor.course.service.impl.CourseGenerateServiceImpl`

### 功能
- 根据知识点生成结构化课程
- 课程内容包含课时和练习
- 学习目标和前置知识
- 双语课程内容（英语/法语）

### 课程生成流程

```
1. 选择知识点的课程范围
2. AI 生成课程大纲：
   - 学习目标
   - 课时结构
   - 练习题
   - 评估标准
3. 课程内容存储（含翻译支持）
4. 学生可按结构化路径学习
```

### 翻译服务
- **集成点**：`com.yy.my_tutor.course.service.impl.CourseTranslationService`
- 自动翻译课程内容（英语 ↔ 法语）
- 保留数学符号和格式

## 4. 题库管理

### 架构演进

| 阶段 | 时间 | 方案 |
|------|------|------|
| v1 | 2025-10 ~ 2026-02 | 定时任务补充题库 |
| v2 | 2026-02+ | 事件驱动补充题库 |

### 事件驱动架构（当前方案）
- **触发条件**：某知识点题目数量低于阈值
- **动作**：发布事件 → 监听器触发 AI 生成
- **优势**：实时补充，无需等待定时任务

```
题库不足 → 发布事件 → QuestionPoolFillJob（监听器）
                              ↓
                      AI 生成新题目
                              ↓
                      题目保存到题库
```

## 5. AI Prompt 工程

### 出题 Prompt 结构
```
系统：你是一位数学教育专家。请根据以下条件生成题目：
- 知识点：{point_name}
- 难度等级：{difficulty_level}
- 题型：{question_type}
- 年级：{grade_name}

输出格式：JSON，包含 question_content, options, correct_answer, answer_explanation
```

### 课程生成 Prompt 结构
```
系统：你是一位课程设计师。请为以下内容创建结构化课程：
- 知识点列表：{point_list}
- 年级：{grade_name}
- 语言：{locale}

输出格式：JSON，包含课程大纲、课时、练习、评估
```

### 对话辅导 Prompt 结构
```
系统：你是一位耐心的数学辅导老师。帮助学生理解概念。
- 使用分步解释
- 适时提供示例
- 鼓励学生
- 语言：匹配学生的语言偏好
```

## 6. AI 模型性能与优化

### 响应时间目标
| 功能 | 目标 | 实际 |
|------|------|------|
| 对话响应（首 Token） | < 2s | ~1.5s |
| 出题（10道题） | < 30s | ~20s |
| 课程生成 | < 60s | ~45s |

### 优化策略
- **流式传输**：对话响应逐 Token 流式输出
- **缓存**：高频生成的题目模式缓存
- **异步处理**：邮件和非关键 AI 任务异步执行
- **上下文窗口**：限制最近 N 条消息，控制 Token 用量
- **批处理**：题库补充按批次处理

### 成本管理
- 每次 API 调用监控 Token 用量
- 题库补充限流，防止过多 API 调用
- 缓存减少相似查询的重复 AI 调用
