# 支付接口文档

本文档整理自 `docs/payment-local-test-curl.md` 和支付模块 Controller,用于前后端联调与接口测试。

## 基础信息

- 本地服务地址: `http://localhost:9009`
- 响应格式:

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- 认证: 正常环境请求头使用 `Authorization: Bearer <token>`。
- 本地测试如开启 `stripe.local-auth-bypass.enabled=true`,可不传认证头。
- 金额字段 `unitAmount`、`amount` 均为最小货币单位,例如 CAD `$9.99` 传 `999`。
- 年度授权目前使用默认卡直接付款,不走 Stripe Checkout;Stripe 按后端计算后的 `PaymentIntent.amount` 扣款。

## 枚举说明

| 字段 | 可选值 |
| --- | --- |
| `productType` | `ANNUAL_LICENSE`, `SUBSCRIPTION_PLAN`, `COURSE`, `KNOWLEDGE_PACK`, `TEST_PACK`, `ADDON` |
| `billingInterval` | `null`=一次性, `month`, `quarter`, `year` |
| `order.status` | `PENDING`, `PAID`, `FAILED`, `REFUNDED`, `PARTIALLY_REFUNDED`, `EXPIRED` |
| `subscription.status` | `incomplete`, `incomplete_expired`, `trialing`, `active`, `past_due`, `canceled`, `unpaid`, `paused` |
| `status` | 商品/价格上下架状态: `1`=上架, `0`=下架 |
| `refund.reason` | `duplicate`, `fraudulent`, `requested_by_customer` |

## 1. 商品接口

### 1.1 查询商品列表

```http
GET /api/payment/products
```

Query 参数:

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `type` | string | 否 | 商品类型,例如 `COURSE` |
| `currency` | string | 否 | 价格币种,例如 `usd` |

curl:

```bash
curl 'http://localhost:9009/api/payment/products?type=COURSE&currency=usd'
```

响应 `data`: `ProductDTO[]`。

### 1.2 查询商品详情

```http
GET /api/payment/products/{id}
```

curl:

```bash
curl 'http://localhost:9009/api/payment/products/1'
```

响应 `data`: `ProductDTO`。

### 1.3 后台创建商品

```http
POST /api/admin/payment/products
```

Body:

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `productType` | string | 是 | 商品类型 |
| `targetRefId` | integer | 否 | 业务实体 ID;订阅商品通常为 `null` |
| `name` | string | 是 | 英文名称 |
| `nameZh` | string | 否 | 中文名称 |
| `nameFr` | string | 否 | 法文名称 |
| `description` | string | 否 | 描述 |
| `prices` | array | 是 | 至少一条价格 |
| `prices[].currency` | string | 是 | 币种,例如 `usd` |
| `prices[].unitAmount` | long | 是 | 最小货币单位金额 |
| `prices[].billingInterval` | string/null | 否 | `null`=一次性;订阅传 `month/quarter/year` |

curl:

```bash
curl -X POST 'http://localhost:9009/api/admin/payment/products' \
  -H 'Content-Type: application/json' \
  -d '{
    "productType": "COURSE",
    "targetRefId": 1,
    "name": "Test Course",
    "nameZh": "测试课程",
    "nameFr": "Cours de test",
    "description": "Local Stripe payment test course",
    "prices": [
      {
        "currency": "usd",
        "unitAmount": 999,
        "billingInterval": null
      }
    ]
  }'
```

响应 `data`: `ProductDTO`。创建 Checkout 时使用 `data.prices[0].id` 作为 `priceId`。

年度授权商品创建说明:

- `productType` 使用 `ANNUAL_LICENSE`。
- 商品只需创建一条 CAD 一次性基础价格。
- 阶梯成交单价维护在 `payment_price_tier` 表,挂在基础 `payment_price.id` 下。
- 测试阶段可配置阶梯单价: `1-5=100`, `6-30=80`, `31+=50`。
- 正式阶段可配置阶梯单价: `1-5=15000`, `6-30=12000`, `31+=10000`。

年度授权测试商品示例:

```bash
curl -X POST 'http://localhost:9009/api/admin/payment/products' \
  -H 'Content-Type: application/json' \
  -d '{
    "productType": "ANNUAL_LICENSE",
    "targetRefId": null,
    "name": "Annual License",
    "nameZh": "年度授权",
    "description": "Annual student activation license",
    "prices": [
      {
        "currency": "cad",
        "unitAmount": 100,
        "billingInterval": null
      }
    ]
  }'
```

### 1.4 后台更新商品

```http

```

Body 字段均可选:

| 字段 | 类型 | 说明 |
| --- | --- | --- |

```bash
    "name": "Updated Course",
```

```http

```

Body:

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `currency` | string | 是 | 币种 |
| `unitAmount` | long | 是 | 最小货币单位金额 |
| `billingInterval` | string/null | 否 | 一次性为 `null`,订阅传 `month/quarter/year` |

### 1.6 后台启停价格

```http

```

Body:

```json

  "status": 1
}
```

curl:

```bash
curl -X PUT 'http://localhost:9009/api/admin/payment/prices/1/status' \
```

响应 `data`: `PaymentPrice`。

## 2. Checkout 与订单接口

### 2.1 创建 Checkout

```http
POST /api/payment/checkout
```

Body:

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `priceId` | integer | 是 | 本地价格 ID |
| `beneficiaryStudentId` | integer | 是 | 获权学生 ID |
| `successUrl` | string | 是 | Stripe 支付成功后跳转地址 |
| `cancelUrl` | string | 是 | Stripe 取消支付后跳转地址 |

curl:

```bash
curl -X POST 'http://localhost:9009/api/payment/checkout' \
  -H 'Content-Type: application/json' \
  -d '{
    "priceId": 1,
    "beneficiaryStudentId": 1,
    "successUrl": "http://localhost:9009/payment/success",
    "cancelUrl": "http://localhost:9009/payment/cancel"
  }'
```

响应 `data`:

```json
{
  "sessionId": "cs_test_xxx",
  "url": "https://checkout.stripe.com/c/pay/cs_test_xxx",
  "orderNo": "ORD20260613xxxx"
}
```

说明:

- `url` 是 Stripe Checkout 托管支付页。
- 支付结果以后端 webhook 为准,不依赖 `successUrl` 页面是否存在。
- 测试卡: `4242 4242 4242 4242`,任意未来有效期,任意 CVC。

### 2.2 查询订单状态

响应 `data`:

### 2.4 查询订单详情


| `order` | object | `PaymentOrder` |

## 3. 退款接口

### 3.1 后台创建退款

```http
POST /api/admin/payment/orders/{orderNo}/refunds
```

Body:

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `amount` | long/null | 否 | `null` 表示全额退款;部分退款传最小货币单位金额 |
| `reason` | string | 是 | `duplicate/fraudulent/requested_by_customer` |
| `reasonDetail` | string | 否 | 退款原因备注 |

全额退款:

```bash
curl -X POST 'http://localhost:9009/api/admin/payment/orders/ORDER_NO/refunds' \
  -H 'Content-Type: application/json' \
  -d '{
    "amount": null,
    "reason": "requested_by_customer",
    "reasonDetail": "Local full refund test"
  }'
```

部分退款:

```bash
curl -X POST 'http://localhost:9009/api/admin/payment/orders/ORDER_NO/refunds' \
  -H 'Content-Type: application/json' \
  -d '{
    "amount": 200,
    "reason": "requested_by_customer",
    "reasonDetail": "Local partial refund test"
  }'
```

响应 `data`: `PaymentRefund`。

### 4.8 默认卡直接付款

```http
POST /api/payment/direct-pay
```

Body:

```json
{
  "priceId": 1,
  "beneficiaryStudentId": 1
}
```

响应 `data`:

```json
{
  "orderNo": "ORD202606150001",
  "paymentIntentId": "pi_xxx",
  "subscriptionId": null,
  "status": "succeeded",
  "clientSecret": "pi_xxx_secret_xxx",
  "requiresAction": false

```

### 4.9 年度授权报价

前端选择年度授权数量后调用该接口展示动态价格。最终支付金额仍以后端直付时重新计算为准,前端展示金额不可作为支付依据。

```http
GET /api/payment/annual-license/quote
```

Query 参数:

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `productId` | integer | 是 | 年度授权商品 ID,商品类型必须为 `ANNUAL_LICENSE` |
| `quantity` | integer | 是 | 购买数量 |

curl:

```bash
curl 'http://localhost:9009/api/payment/annual-license/quote?productId=1&quantity=10'
```

响应 `data`:

```json
{
  "productId": 1,
  "priceId": 1,
  "priceTierId": 2,
  "currency": "cad",
  "quantity": 10,
  "unitAmount": 80,
  "totalAmount": 800
}
```

说明:

- `unitAmount` 为命中阶梯后的每个授权成交单价。
- `totalAmount = unitAmount * quantity`。
- 当前测试价格示例: 买 10 个命中 `6-30` 档,单价 CAD `$0.80`,总价 CAD `$8.00`。

### 4.10 默认卡直接购买年度授权

年度授权复用 `POST /api/payment/direct-pay`,请求带 `productId` 或 `quantity` 时进入年度授权购买逻辑。

```http
POST /api/payment/direct-pay
```

Body:

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `productId` | integer | 是 | 年度授权商品 ID |
| `quantity` | integer | 否 | 购买数量。学生固定只能买 1 个;老师可买多个 |

学生购买示例:

```bash
curl -X POST 'http://localhost:9009/api/payment/direct-pay' \
  -H 'Content-Type: application/json' \
  -d '{
    "productId": 1,
    "quantity": 1
  }'
```

老师购买示例:

```bash
curl -X POST 'http://localhost:9009/api/payment/direct-pay' \
  -H 'Content-Type: application/json' \
  -d '{
    "productId": 1,
    "quantity": 10
  }'
```

响应 `data`: `DirectPaymentResponse`。

年度授权直付规则:

- 学生只能购买 `quantity=1`,支付成功后从激活日期开始设置一年有效期。
- 老师购买时不选择受益学生,支付成功后增加老师可用年度授权名额。
- 老师给学生使用名额发生在师生绑定接口,见 `POST /api/guardian-rel/bind?activate=true`。
- 后端按 `payment_price_tier` 命中的单价计算最终金额,再把总金额传给 Stripe `PaymentIntent.amount`。

### 4.11 默认卡直接创建订阅

## 5. Stripe Webhook

### 5.1 接收 Stripe Webhook

```http
POST /api/payment/stripe/webhook
```

Header:

| Header | 必填 | 说明 |
| --- | --- | --- |
| `Stripe-Signature` | 是 | Stripe 签名头 |

说明:

- 该接口由 Stripe 或 Stripe CLI 调用,业务前端不直接调用。
- 本地测试使用:

```bash
stripe listen --forward-to localhost:9009/api/payment/stripe/webhook
```

- 成功返回 `200 ok`。
- 缺少签名返回 `400 missing signature`。
- handler 处理失败返回 `500 handler failed`,Stripe 会重试投递。

## 6. 本地测试推荐流程

1. 启动后端服务。
2. 启动 Stripe CLI webhook 转发。
3. 后台创建课程商品、订阅商品或年度授权商品。
4. 查询商品列表,记录 `priceId`。
5. 创建 Checkout,打开返回的 `url` 支付。
6. 查询订单状态,确认从 `PENDING` 变为 `PAID`。
7. 查询订单详情和订阅列表。
8. 如需退款,调用后台退款接口后再查询订单详情。
